# CHE110_Project

This project focuses on presenting educational content through multiple web pages, covering topics such as ecosystem types, benefits, hazards, prevention, and related environmental concepts. It includes SEO optimization, proper folder structure, and responsive layouts to enhance user experience.

## Live Link
- **CHE110_Project:** [View CHE110 Project](https://pr2309.github.io/CHE110_Project/)

## Development Server

To view the CHE110 Project locally:
1. Clone the repository.
2. Open the `index.html` file located **at the root** of the project in your browser.

## Project Structure
```
CHE110_Project/
│
├── index.html               # Homepage (root)
├── sitemap.xml              # Sitemap added for SEO
│
├── Pages/                   # All subpages such as Benefits, Hazards, Types, etc.
│     ├── benefits.html
│     ├── hazards.html

│     ├── types.html

│
└── Style/                   # All CSS files
      ├── style.css

      └── ... other styling files
```

## Features
- Multi-page static website for ecosystem education.
- Internal navigation through buttons and headers.
- Responsive and mobile-friendly design.
- Parallel image folder and style folder for maintainability.
- SEO-optimized head tags on all pages.
- Sitemap integration for better search engine crawling.

## How to Use
1. Open the `index.html` file located in the **root** directory.
2. Navigate to different pages from the homepage.
3. Internal navigation allows:
   - Jumping between levels (1–6)
   - Moving between ecosystem types (forest, grassland, tundra, marine, etc.)
   - Switching between pages like Benefits, Hazards, Prevention, and About.
4. All pages are stored inside the `Pages/` folder.

## SEO Enhancements Added
The project includes multiple search-engine-friendly improvements:
- Canonical URL tag  
- Robots meta tag  
- Description + keywords  
- Open Graph tags for social media previews  
- Twitter card tags  
- Alternate hreflang tags  
- Favicon support for multiple devices  
- `sitemap.xml` for indexability  

These optimizations help GitHub Pages index your static site correctly.
